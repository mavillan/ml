El Problema¶

El problema consiste en la correcta identificacin y reconocimiento facial, dado el caso particular en donde existen muy pocas muestras por clase (por persona) para entrenar los algoritmos algoritmos de clasificacin. La mayora de los enfoques usados en este tipo de problemas consisten en encontrar una buena representacin de las caractersticas importantes de las caras, para posteriormente realizar algn tipo de bsqueda (jerrquica, nearest neighbors, etc). Sin embargo se propone aqu resolver el problema con tres algoritmos de clasificacin distintos: Linear Discriminant Analysis, Support Vector Machines (con kernel lineal y radio basal) y por Convolutional Neural Networks.

Los dataset a ocupar son Faces94, Faces95 y Faces96. Cada uno de los datasets consiste en 20 imgenes de individuos, y variable cantidad de individuos. Los datasets estn ordenados en cuanto a su complejidad de reconocimiento de menor a mayor. Se muestran a continuacin imgenes representativas de Faces94, Faces95 y Faces96 respectivamente.

Para cada uno de estos datasets, se crearon los 20 training y testing sets correspondientes. La metodologa fue la siguiente: Para cada dataset (94,95,96) se tomaron aleatoriamente entre 2-5 fotos por clase (por persona) para formar los training sets correspondientes, y las restantes 15-18 fotos se dejaron para crear los testing sets correspondientes. Vale decir, si se entrena con train94/5pc (faces94, 5 samples per class) entonces se prueba con test94/15pc (faces94, 15 samples per class). Cumpliendo de este modo con la restriccin de tener pocos samples para entrenar los algoritmos. Observacin: Por el momento se trabaja slo con Faces94 y Faces95.

Para la comparacin entre los resultados de los distintos algoritmos, se realizan Error Bars del error rate sobre los 20 datasets. Para computar las barras de error, se computan segn el intervalo de confianza del 95% para la media del error, por medio de la distribucin t-student.

La implementacin ocupada corresponde la de Scikit-Learn. Para cada una de las clases (personas) genera la funcin discriminante lineal $\delta_k$, que permite diferenciar a cada una de las clases. Los dos supuestos fuertes que se realizan sobre los datasets al ocupar este mtodo, son que 1) La probabilidad multivariada de las caractersticas $P(x_m | y=k)$ se distribuye normal, y que 2) La matriz de covarianza para cada una de las clases es igual.

Debido a que LDA es un modelo generativo sin hiperparmetros, es que no es necesario realizar cross-validation para el modelo. Esto es una gran ventaja, pues ahora gran tiempo de computacin (comparado con los otros mtodos). Sin embargo, como se ver ms adelante, paga este costo de simplicidad, entregando resultados menos precisos y generalizantes.

Enfoque 2: Support Vector Machines¶

Marco Teórico¶

Support Vector Machines (SVM) es un mtodo de clasificacin binario (adaptable a problemas de mltiples clases), que encuentra la frontera de decisin lineal ptima (hiperplano ptimo) que separa a las clases. Intuituvamente, una buena separacin es lograda por el hiperplano con mayor distancia o margen a los datos de entramiento ms cercanos. Esta superficie de separacin es una combinacin lineal de elementos del training set, conocidos como vectores de soporte, pues definen la frontera entre dos clases

Por lo tanto el mtodo se reduce a el problema de optimizacin, de encontrar el hiperplano $\mathbf{w}^T x + b = 0$ que maximice el margen de cada ejemplo en el training set. Esto puede expresarse mas formalmente como: \begin{align} & \min_{\mathbf{w},b} \frac{1}{2} ||\mathbf{w}||^2 \ & \text{s.t } \ y_m(\mathbf{w}^T x_m + b) \geq 1, \ m=1,\ldots ,M \end{align}

lo cual corresponde a un problema de optimizacin cuadrtico. Es posible relajar algunas reestricciones del problema introduciendo variables de holgura \begin{align} & \min {\mathbf{w}, b, \zeta} \frac{1}{2} ||\mathbf{w}||^2 + C \sum{m=1}^{M} \zeta_i \ & \text{s.t } \ y_m (\mathbf{w}^T \ x_m + b) \geq 1 - \zeta_m,\ & \zeta_m \geq 0, m=1, ..., M \end{align}

donde el parmetro C controla el trade-off entre el error de clasificacin y el tamao del margen. En general a mayor margen, menor ser el error de generalizacin del clasificador, por lo tanto con las variables de holgura se tiene una manera de controlar esta relacin.

La SVM determinada por el anterior problema de optimizacin es conocida como la C-SVM. Sin embargo por medio de una re-parametrizacin de esta, es posible definir un nuevo parmetros $\nu$ que controla el nmero de vectores de soporte y el error de entrenamiento. Esta formulacin corresponde a la $\nu$-SVM y puede ser demostrado que ambas formulaciones son matemticamente equivalentes.

Experimentación¶

El enfoque seguido en esta seccin, es la implementacin y entrenamiento de multiclass SVM's con kernels tanto lineales como gaussianos, para cumplir con el objetivo de clasificar la data. Para ello se ocupan $\nu$-SMV's, debido a que facilita la configuracin del parmetro de holgura $\nu$. La implementacin ocupada corresponde a la de Scikit-Learn, el cual advierte: SVC implement one-against-one approach (Knerr et al., 1990) for multi- class classification."

Para la seleccin de hiperparmetros $\nu$ y $\gamma$ (en kernels rbf) se realiza stratified cross-validation sobre cada training set, con la ayuda de grid search. Para que este mtodo tenga sentido, el nmero de folds debe ser igual al nmero de muestras por clase (si hay ms folds que muestras por clase, no se pueden cumplir las condiciones de estratificacin).

Debido a que las dimensiones de las imgenes (200x180=36000) corresponden al total de features de cada foto, se ha decidido realizar una reduccin de dimensionalidad para mejorar los tiempos de entrenamientos y eficiencia, as como tambin tomar las caractersticas realmente importantes (aquellas que permiten diferencias entre las clases).

Como tcnica de reduccin de dimensionalidad, se ha decidido ocupar LDA como reduccin de dimensionalidad supervisada, representacin tambien conocida como Fisher Faces. Esta tcnica intenta encontrar un subespacio donde proyectar la data que permita diferenciar de mejor manera las clases. Dicho de otro modo, se intenta maximizar la inter-class variance y minimizar la varianza intra-class variance.

Bsicamente los hiperplanos que conforman el subespacio de representacin, corresponden a la funciones discriminantes que genera el modelo en LDA. La idea se plasma en la siguiente representacin

en donde cada dato se proyecta en los hiperplanos discriminantes, para formar la representacin. El nmero de mximo de discriminantes es $\min(\text{dimensiones},\text{clases}-1)$, por lo tanto en un problema con muchas ms dimensiones (features) que clases, la reduccin de dimensionalidad es considerablemente favorable (este problema es precisamente el caso). Se pueden ocupar menos discriminantes, pero para los experimentos que se muestran a continuacin se ocupan todos.

Enfoque 3: Convolutional Neural Network¶

Marco teórico¶

Las redes neuronales convolucionales son un algoritmo de amplia aceptacin hoy en dia, para tareas relacionadas con el reconocimiento de patrones, principalmente en imgenes. Estas nacen bajo el concepto de que los pxeles cercanos dentro de una imagen estn ms relacionados entre s, que con los lejanos, y por lo tanto es necesario tomar en cuenta la informacin espacial de las caractersticas en una imagen.

Estas redes corresponden a una versin extendida de las conocidas redes feedforward, que agrega dos tipos de capas nuevas:

  • Covolucional: Consiste de un arreglo rectangular de neuronas, que requiere que la capa previa sea tambin un arreglo rectangular de inputs. Cada una de estas neuronas toma inputs de una seccin rectangular de la capa previa (llamado local receptive field), siendo los pesos para esta regin rectangular los mismos para cada neurona en la capa convolucional (weight sharing). Entonces, la capa convolucional puede entenderse como una operacin de convolucin de una matriz de kernel/filtro (con los pesos) con la imagen de la capa previa. La siguiente imagen ilustra tal concepto: Intuitivamente, la capa convolucional al aplicar el mismo kernel/filtro a toda la imagen, lo que est haciendo es computar la misma caracterstica a lo largo de tal imagen, por lo tanto el output de una red convolucional es llamado feature map, pues es un mapa que indica donde esta o no presente tal caracterstica. Usualmente varias capas convolucionales operan en paralelo, cada una computando una caracterstica distinta y generando el feature map correspondiente.

  • Pooling: Despus de cada capa convolucional, suele estar una capa de pooling. Estas toman una pequea regin rectangular desde el el output de la capa convolucional (usualmente ms pequeo que el tamao del filtro convolucional), y realiza subsampling, generando un solo input desde el bloque. La operacin realizada por pooling puede implementarse de varias formas: Tomando el mximo de los valores, el promedio, promedio con pesos, etc. La siguiente imagen ilustra el concepto de max pooling: El objetivo del pooling es dar holgura espacial a la posicin de las caractersticas computadas, de tal modo que si las caractersticas de una determinada imagen se ven trasladadas, el resultado de la red convolucional no se vea afectado.

La arquitectura de la red utilizada vara segn la aplicacin y los propsitos, pero en general suele utilizarse varias capas de Convolucin-Pooling juntas, permitiendo extraer caractersticas cada vez ms abstractas.

Despus de cualquier configuracin de capas convolucional y de pooling, el aprendizaje de alto nivel de la red, sigue siendo por una capa densa (fully connected) seguida por una capa softmax si se est realizando clasificacin. Una configuracin tpica se muestra en la imagen a continuacin:

Enfoque 4: Dissimilarity SVM¶

Marco Teórico¶

Representación¶

El problema de identificacin tratado corresponde a dada una imagen de prueba $\mathbf{p}$, determinar a qu clase del conjunto de entrenamiento $S_{tr}$ corresponde. Los enfoques basado en SVM anteriores, reducen este problema a $K$-class classification.

El propsito del mtodo planteado, es ocupar las capacidades de clasificacin de las SVM como clasificador binario, sobre dos conjuntos $C_1$ y $C_2$, donde el primero corresponde al within-class differences set que contienen las disimilitudes entre datos de la misma clase, y el segundo es between-class difference set y contienen las disimilitudes entre datos de distinta clase. El espacio en el que habitan los elementos de estos conjuntos, es conocido como el difference space, y contrasta con el espacio standard de las imgenes conocido como image space.

Formalizando lo anterior; sea $S_{tr} = \{\mathbf{s}_1 , \ldots, \mathbf{s}_M \}$ el conjunto de entrenamiento con imgenes faciales de $K$ individuos. Para indicar que dos individuos pertenecen a la misma clase ocupamos $\mathbf{s}_i \sim \mathbf{s}_j$, y en caso contrario $\mathbf{s}_i \nsim \mathbf{s}_j$. Se define adicionalmente la funcin de similitud $\phi : R^N \times R^N \rightarrow R^S$ con $S \leq N$, como aquella funcin que mapea dos images, hacia el difference space. Luego es posible definir \begin{align} & C_1 = {\phi(\mathbf{s}_i, \mathbf{s}_j)\ | \ \mathbf{s}_i \sim \mathbf{s}_j } \ & C_2 = {\phi(\mathbf{s}_i, \mathbf{s}_j)\ | \ \mathbf{s}_i \nsim \mathbf{s}_j } \end{align}

Entrenamiento¶

Para el entrenamiento de la D-SVM (Dissimilarity SVM) los conjuntos de entrada son $C_1$ y $C_2$, es decir, se realiza un simple entrenamiento para clasificacin binaria, lo cual es bastante conveniente.

Adicionalmente, sabemos que el output de una SVM es un conjunto de $M_s$ vectores de soporte $\mathbf{v}_m$, coeficientes $\alpha_m$, etiquetas de las clases $y_m$ de los vectores de soporte y el trmino $b$ constante. Luego la superficie de decisin puede ser escrita como $$ f(\mathbf{x}) = \sum_m^{M_s} \alpha_m y_m K(\mathbf{v}_m, \mathbf{x}) + b = 0 $$

donde $K(\cdot,\cdot)$ es una funcin de kernel de acuerdo al Mercer's Theorem. Para $f(\mathbf{x}_i)<0$, mientras ms grande sea el valor de $|f(\mathbf{x_i})|$, ms es el grado (o probabilidad) de pertenencia de $\mathbf{x}_i$ a la primera clase (de modo anlogo se concluye para la segunda clase). Por lo tanto se puede ocupar la funcin $f$ como discriminante, o como score de pertenencia a una clase.

Clasificación¶

El proceso de identificacin es como sigue:

  • Sea $\mathbf{p}$ la imagen a identificar sobre un conjunto de imgenes conocidas $S_{tr}$. Se computa la representacin en el difference space: $\phi(\mathbf{p},\mathbf{s_i}) \ \forall \mathbf{s_i} \in S_{tr}$.
  • Evaluar la funcin de discriminante $f(\mathbf{x})$ sobre cada uno de los resultados anteriores.
  • La clase de pertenencia ser aquella para la cual $f(\phi(\mathbf{p},\mathbf{s_i}))$ sea la menor posible: \begin{align} 
      \underset{i} {\mathrm{argmin}} ~f(\phi(\mathbf{p},\mathbf{s_i}))
    \end{align}

Local Binary Pattern¶

Esta es una forma de representacin de imgenes, como una distribucin de micro-patrones que describen las estructuras y formas presentes en estas. Los distintos patrones que emergen, corresponden a las caractersticas presentes en las imgenes, y por lo tanto la distribucin de las caractersticas en una imagen, sirven de referencia para poder compararlas.

La idea de bsica de LBP como operador, se resumen en la siguiente imgen: el operador lo que hace es comparar a cada pxel con el valor de sus vecinos, generando un cdigo binario que indica si el vecino tiene mayor o menor magnitud respectivamente. Tal operador se convoluciona en toda la imagen generando un patrn por cada pixel. En Face description with local binary pattenrs, Ahonen et al. se argumenta que no todos los patrones son realmente tiles en cuanto a informacin. Nace entonces el concepto de uniform pattern como aquel que no tiene ms de una transicin cclica de 0 a 1 o viceversa. Tales patrones son como los de la figura: que corresponden a 58 patrones. Adicionalmente se agrega un patrn 59 en el que entran todas las otras caractersticas no uniforme. Entonces cada pxel puede ser codificado con un patron de 59 bits.

El mismo autor propone luego la idea de ocupar histograma de caractersticas con estos patrones, en donde cada bin mide la frecuencia de una caracterstica, y la distribucin resultante de los histogramas definen entonces las estructuras presentes en las imgenes. Sin embargo es necesario agregar informacin espacial, acerca de donde se encuentran estas caractersticas, por lo tanto proponen segmentar las imgenes en regiones rectangulares, computando un histograma de caractersticas para cada una de estas regiones y luego concatenar los histogramas resultantes, para obtener la distribucin resultante que representa a una persona.

Método propuesto¶

El modelo propuesto difiere de aquel propuesto por P. Jonathon Philips en Support Vector Machines Applied to Face Recognition, en la funcin $\phi(\cdot,\cdot)$ de mapeo al difference space. Sean dos imgenes representadas en la forma de histograma espacial $(\mathbf{h_1},\mathbf{h_2})$ con $S$ histogramas, luego la funcin que se propone es: $\phi(\mathbf{h_1}, \mathbf{h_2}) = \mathbf{d} = (d_1, \cdots, d_S)$ tal que $\mathbf{d_i}$ corresponde a la diferencia existente entre los histogramas $i$-simos de las respectivas imgenes. Tal diferencia se computa con la funcin: $$\chi^2(x,y) = \sum_k \frac{(x_k - y_k)^2}{x_k + y_k}$$ conocida por ser una buena mtrica para comprar histogramas. Aunque sera interesante probar otras variantes como interseccin de histogramas.

Detalles de implementación¶

  • Cuando existen $s$ muestras por clase, se pueden hacer $\binom{s}{2}$ combinaciones para formar el conjunto $C_1$.
  • El conjunto $C_2$ se forma tomando aleatoriamente igual cantidad ($\binom{s}{2}$) de pares de datos de distinta clase.
  • El valor que obtiene la funcin discriminante $f(\mathbf{x})$, se promedia para los datos que pertenecen a la misma clase en $S_{tr}$

"""
> function to load data from path directory to a matrix.
> each row of the resulting matrix, corresponds to a flattened image
  in grayscale format

def load_data(path, stacked=False):
    #total number of classes
    M = len(os.listdir(path))
    #dimensions of each image
    N = 200*180
    #samples per class
    spc = int(path.strip().split('-')[1][:-2])
    #matrix with features
    if stacked:
        data = np.empty((M*spc,200,180), dtype=np.float32)
    else: 
        data = np.empty((M*spc,N), dtype=np.float32)
    labels = np.empty(M*spc, dtype=np.uint8)
    #index of data matrix
    m = 0
    for i in xrange(1,M+1):
        tgt = path+str(i)+'/'
        pics = os.listdir(tgt)
        for pic in pics:
            if stacked:
                #store each image, as a bidimensional array in data matrix
                data[m,:,:] = cv.imread(tgt+pic, cv.IMREAD_GRAYSCALE)
            else:
                #store each flattened image, as a row in data matrix
                data[m,:] = cv.imread(tgt+pic, cv.IMREAD_GRAYSCALE).ravel()
            labels[m] = i
            m += 1
    return (data, labels)

def cross_linear_svm(X, y, Nu, n_folds=None):
    #generating stratified 5-fold cross validation iterator
    strat_kf = StratifiedKFold(y, n_folds=n_folds, shuffle=True)
    #parameters to try
    params = {'nu':Nu}
    #Setting grid search for linear-svm

    gs.fit(X, y)

    return gs.best_params_['nu'] , gs.grid_scores_

    #generating stratified 5-fold cross validation iterator
    strat_kf = StratifiedKFold(y, n_folds=n_folds, shuffle=True)

    params = {'nu':Nu, 'gamma':Gamma}

    #make it
    gs.fit(X, y)
    #return best parameters and grid scores
    return gs.best_params_['nu'], gs.best_params_['gamma'] , gs.grid_scores_

#chi2 distance metric
def chi2(h0, h1, tol=1e-10):
    if h1.ndim==1:
        return np.sum((h0-h1)**2./((h0+h1)+tol))
    elif h1.ndim==2:
        return np.sum((h0-h1)**2./((h0+h1)+tol),axis=1)

class dSVM(BaseEstimator, ClassifierMixin):
    def __init__(self, P=8, R=2, N=8, OverlapX=2, OverlapY=2, nu=0.1, gamma=0.5):
        self.P = P #number of neighbors to compute LBP
        self.R = R #radius to compute LBP
        self.N = N #Number of partitions on x-axis and y-axis (spatial histogram)
        self.Npatterns = 59 #number of bins of each histogram (not settable)
        self.OverlapX = OverlapX #allowed overlap between regions on x-axis
        self.OverlapY = OverlapY #allowed overlap between regions on y-axis
        self.nu = nu #nu parameter for Nu-SVM
        self.gamma = gamma #gamma parameter for kernel on SVM
    """
    Transform data on X matrix (flattened) to 
    spatial histograms representation on Y matrix
    """
    def to_spatial_histogram(self, X):
        M,N = X.shape
        #return matrix
        Y = np.empty((M,self.Npatterns*self.N**2))
        for m in xrange(M):
            img = X[m].reshape((-1,180))
            lbp_img = lbp(img, self.P, self.R, method='nri_uniform')
            lbp_img = lbp_img.astype(np.uint8)
            sp_hist = histogram.spatial(lbp_img, self.N, self.N,
                      self.Npatterns, self.OverlapX, self.OverlapY)
            Y[m] = sp_hist
        return Y

    #compute frecuency difference
    #def dissimilarity1(h0, h1):
    #    return np.abs(h0-h1)

    """
    > computes dissimilarity between h0 and h1
    > h1 could be a 2D array. In that case
      dissimilarities are computed between h0 and
      each row in h1
    """
    def dissimilarity(self, h0, h1):
        if h1.ndim==1:
            N = h0.shape[0]
            #number of histograms (dimensions on difference space)
            S = N/59
            diff = np.empty(S)
            s = 0 #index of diff
            for j in xrange(0,N,59):
                diff[s] = chi2(h0[j:j+59],h1[j:j+59])
                s += 1
            return diff
        elif h1.ndim==2:
            M,N = h1.shape
            #number of histograms (dimensions on difference space)
            S = N/59
            diff = np.empty((M,S))
            s = 0 #column index of diff
            for j in xrange(0,N,59):
                diff[:,s] = chi2(h0[j:j+59],h1[:,j:j+59])
                s += 1
            return diff

    def build_classes(self, hist_matrix, y, spc):
        #dimensions of input matrix
        M,N = hist_matrix.shape
        #indexes of hist_matrix
        indexes = np.arange(M)
        np.random.shuffle(indexes)
        #dimensions of output matrices
        P = int(sp.misc.comb(spc,2))*152
        Q = self.N**2
        """
        > building the within class difference set C1
        > all combinations between samples of the same
          class are computed
        """
        C1 = np.empty((P,Q))
        p = 0 #index of C1
        for m in xrange(0, M, spc):
            for i in xrange(m, m+spc):
                for j in xrange(i+1, m+spc):
                    #difference space representation
                    C1[p] = self.dissimilarity(hist_matrix[i], hist_matrix[j])
                    p += 1
        """
        > building the between class difference set
        > randomly selecting two samples of different classes
        """
        C2 = np.empty((P,Q))
        p = 0 #index of C2
        while p < P:
            i,j = np.random.choice(indexes, 2)
            #indexes must be of different classes
            if y[i]==y[j]: continue
            C2[p] = self.dissimilarity(hist_matrix[i],hist_matrix[j])
            p += 1
        #return both classes
        return (C1,C2)

    def fit(self, X, y):
        #counting samples per class
        frec = np.bincount(y-1)
        spc = frec[0]
        #if any class has different frec
        if not np.all(spc==frec): return -1
        
        #X data to spatial histogram format
        hist_matrix = self.to_spatial_histogram(X)
        
        #building classes
        C1,C2 = self.build_classes(hist_matrix,y,spc)
        data = np.concatenate((C1,C2), axis=0)
        #label arrays
        lab1 = np.ones(C1.shape[0],dtype=int)
        lab2 = 2*lab1
        labels = np.concatenate((lab1,lab2))
        
        #building svm with stratified cross validation
        strat_kf = StratifiedKFold(labels, n_folds=5, shuffle=True)
        #parameters to try
        Nu = np.linspace(0.1, 0.5, 5, endpoint=True)
        Gamma = np.linspace(0.25, 1.5, 5, endpoint=True)
        params = {'nu':Nu, 'gamma':Gamma}
        #grid search
        clf = svm.NuSVC(kernel='rbf')
        gs = grid_search.GridSearchCV(clf, params, cv=strat_kf, n_jobs=2)
        gs.fit(data,labels)
        #fitting with best parameters
        dsvm = svm.NuSVC(kernel='rbf', nu=gs.best_params_['nu'], gamma=gs.best_params_['gamma'])
        dsvm.fit(data,labels)
        
        #storing important things to make predictions
        self.spc = spc
        self.data = hist_matrix
        self.dsvm = dsvm
        return self

    def predict(self, X):
        #X data to spatial histogram format
        X = self.to_spatial_histogram(X)
        M = X.shape[0]
        predictions = np.empty(M, dtype=np.uint8)
        for m in xrange(M):
            diff = self.dissimilarity(X[m],self.data)
            df = self.dsvm.decision_function(diff)
            N = df.shape[0]
            min_val = np.inf
            min_ind = 0
            for i in xrange(0,N,self.spc):
                mean_score = np.sum(df[i:i+self.spc])/self.spc
                if mean_score < min_val:
                    min_val = mean_score
                    min_ind = i/self.spc
            predictions[m] = min_ind
        #shift to match the labels
        predictions += 1
        return predictions
    
    def score(self, X, y):
        y_pd = self.predict(X)
        return precision(y, y_pd)

def solve_dsvm(dataset, spc, verbose=False):
    #samples per class on training and testing set
    spc_tr = spc
    spc_ts = 20-spc_tr
    #training and testing paths
    tr_path = './db/train'+dataset[-2:]+'/tr-{0}pc-{1}/'
    ts_path = './db/test'+dataset[-2:]+'/ts-{0}pc-{1}/'
    #errors through all datasets
    tr_err = list()
    ts_err = list()
    #iterating through datasets
    for set_num in xrange(20):
        #loading training and testing sets
        X_tr,y_tr = load_data(tr_path.format(spc_tr,set_num))
        X_ts,y_ts = load_data(ts_path.format(spc_ts,set_num))
