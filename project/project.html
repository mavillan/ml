Proyecto de Máquinas de Aprendizaje - ILI393¶

Identificación facial cuando existen pocos ejemplos por clase¶

El Problema¶

El problema consiste en la correcta identificacin y reconocimiento facial, dado el caso particular en donde existen muy pocos ejemplos por clase (por persona) para entrenar los algoritmos algoritmos de clasificacin. La mayora de los enfoques usados en este tipo de problemas consisten en encontrar una buena representacin de las caractersticas importantes de las caras, para posteriormente realizar algn tipo de bsqueda (jerrquica, nearest neighbors, etc). Sin embargo se propone aqu resolver el problema con tres algoritmos de clasificacin distintos: Linear Discriminant Analysis, Support Vector Machines (con kernel lineal y radio basal) y por Convolutional Neural Networks.

Los dataset a ocupar son Faces94, Faces95 y Faces96. Cada uno de los datasets consiste en 20 imgenes de individuos, y variable cantidad de individuos. Los datasets estn ordenados en cuanto a su complejidad de reconocimiento de menor a mayor. Se muestran a continuacin imgenes representativas de Faces94, Faces95 y Faces96 respectivamente.

Para cada uno de estos datasets, se crearon training y testing sets correspondientes. La metodologa fue la siguiente: Para cada dataset (94,95,96) se tomaron aleatoriamente entre 2-5 fotos por clase (por persona) para formar los training sets correspondientes, y las restantes 15-18 fotos se dejaron para crear los testing sets correspondientes. Vale decir, si se entrena con train94/5pc (faces94, 5 samples per class) entonces se prueba con test94/15pc (faces94, 15 samples per class). Cumpliendo de este modo con la restriccin de tener pocos samples para entrenar los algoritmos.

Observacin: Por el momento se trabaja slo con Faces94 y Faces95.

Enfoque 1: Linear Discriminant Analysis¶

La implementacin ocupada corresponde la de Scikit-Learn. Para cada una de las clases (personas) genera la funcin discriminante lineal $\delta_k$, que permite diferenciar a cada una de las clases. Los dos supuestos fuertes que se realizan sobre los datasets al ocupar este mtodo, son que 1) La probabilidad multivariada de las caractersticas $P(x_m | y=k)$ se distribuye normal, y que 2) La matriz de covarianza para cada una de las clases es igual.

Debido a que LDA es un modelo generativo sin hiperparmetros, es que no es necesario realizar cross-validation para el modelo. Esto es una gran ventaja, pues ahora gran tiempo de computacin (comparado con los otros mtodos). Sin embargo, como se ver ms adelante, paga este costo de simplicidad, entregando resultados menos precisos y generalizantes.

Enfoque 2: Support Vector Machines¶

El enfoque seguido en esta seccin, es la implementacin y entrenamiento de multiclass SVM's con kernels tanto lineales como gaussianos, para cumplir con el objetivo de clasificar la data. Debido a que los datasets fueron generados de manera balanceada en cuanto a las clases, se utilizan C-SVM's. La implementacin ocupada corresponde a la de Scikit-Learn, la cual advierte: SVC implement one-against-one approach (Knerr et al., 1990) for multi- class classification."

Para la seleccin de hiperparmetros $C$ y $\gamma$ (en kernels rbf) se realiza 5-fold cross validation sobre cada training set, con la ayuda de grid search.

Debido a que las dimensiones de las imgenes (200x180=36000) corresponden al total de features de cada foto, se ha decidido realizar una reduccin de dimensionalidad para mejorar los tiempos de entrenamientos y eficiencia, as como tambin tomar las caractersticas realmente importantes (aquellas que permiten diferencias entre las clases).

Como tcnica de reduccin de dimensionalidad, se ha decidido ocupar LDA como reduccin de dimensionalidad supervisada, representacin tambien conocida como Fisher Faces. Esta tcnica intenta encontrar un subespacio donde proyectar la data que permita diferenciar de mejor manera las clases. Dicho de otro modo, se intenta maximizar la inter-class variance y minimizar la varianza intra-class variance.

Bsicamente los hiperplanos que conforman el subespacio de representacin, corresponden a la funciones discriminantes que genera el modelo en LDA. La idea se plasma en la siguiente representacin

en donde cada dato se proyecta en los hiperplanos discriminantes, para formar la representacin. El nmero de mximo de discriminantes es $\min(\text{dimensiones},\text{clases}-1)$, por lo tanto en un problema con muchas ms dimensiones (features) que clases, la reduccin de dimensionalidad es mayor (este problema es precisamente el caso). Se pueden ocupar menos discriminantes, pero para los experimentos que se muestran a continuacin se ocupan todos.

"""
> function to load data from path directory to a matrix.
> each row of the resulting matrix, corresponds to a flattened image
  in grayscale format
"""
def load_data(path, spc):
    path = path+str(spc)+'pc/'
    #total number of classes
    M = len(os.listdir(path))
    #dimensions of each image
    N = 200*180
    #matrix with features and labels
    data = np.empty((M*spc,N+1))
    #index of data matrix
    m = 0
    for i in range(1,M+1):
        tgt = path+str(i)+'/'
        pics = os.listdir(tgt)
        for pic in pics:
            #store each flattened image, as a row in data matrix
            data[m,0:-1] = cv.imread(tgt+pic, cv.IMREAD_GRAYSCALE).ravel()
            data[m,-1] = i
            m +=1
    #just to verify that orden doesn't affect results
    #and cross-validation purposes
    np.random.shuffle(data)
    return (data[:,0:-1], data[:,-1])

def precision(y_ts, y_pd):
    #true positives
    tp = (y_ts==y_pd).sum()
    #total of predictions
    n = y_ts.shape[0]
    return tp/np.float(n)

def error_rate(y_ts, y_pd):
    return 1-precision(y_ts, y_pd)

"""
5-fold cross validation y Grid search
para determinar el mejor parametro C en
linear svm
"""
def cross_linear_svm(X, y, C):
    #generating 5-folds
    M,_ = X.shape
    kf = KFold(M, n_folds=5)
    #parameters to try
    params = {'C':C}
    #Setting grid search for linear-svm and 5-fold cross validation
    svc = svm.SVC(kernel='linear')
    clf = grid_search.GridSearchCV(svc, params, cv=kf, n_jobs=4)
    #make it
    clf.fit(X, y)
    #return best parameters and grid scores
    return clf.best_params_['C'] , clf.grid_scores_


"""
5-fold cross validation y Grid search
para determinar el mejor parametro C y

def cross_rbf_svm(X, y, C, Gamma):
    #generating 5-folds
    M,_ = X.shape
    kf = KFold(M, n_folds=5)
    #parameters to try
    params = {'C':C, 'gamma':Gamma}
    #Setting grid search for rbf-svm and 5-fold cross validation
    svc = svm.SVC(kernel='rbf')
    clf = grid_search.GridSearchCV(svc, params, cv=kf, n_jobs=4)
    #make it
    clf.fit(X, y)
    #return best parameters and grid scores
    return clf.best_params_['C'], clf.best_params_['gamma'] , clf.grid_scores_
